anonymizeR

anonymizeR is a package that assists in data anonymization by creating easy-to-use methods for data perturbation, generalization, and suppression.

Data Anonymization: What is it and Why is it important?

In data sets, individuals are associated with several attributes, each of which contains some value for that individual. In private data sets, the values of these attributes are often sensitive information, and it could bring harm to the individual or the organization if the values of his attributes were released to outside entities. However, organizations often want to share data with outside entities, or even the public. In order to share data without harming individuals who may have sensitive information stored in the data set, the data set must be cleaned so that it is impossible to determine which individual corresponds to which observation within the data set. This is what data anonymization is all about.

In any data set with variables and observations, variables will fall into one of the following three categories:

  • Identifiers - the values of these variables have a one-to-one or near one-to-one relation with individuals. If an identifier for an individual is known, then it is trival to identify the individual. Examples may include full name, address, social security number, or some other form of unique ID.

  • Key Variables - the values of these variables may not have a one-to-one relation with individuals, but they are unique enough that a combination of them, some which may be outside of the data set in question, can be used to identify an individual similar to an identifier. Some examples of key variables are first name, last name, gender, race, age, and postal code.

  • Non-identifying Variables - the values of these variables are very difficult or impossible to identify an individual with. Anything that is not an identifier or a key variable is a non-identifying variable.

When publishing private data for public use, each of the three variable types must be treated differently. Identifiers should always be removed; they expose too much private information to the public. Non-identifying variables, however, provide no private information and may be kept in.

Key variables are where the problem comes in - the bulk of data analysis typically occurs on key variables, so it is important to keep key variables. However, having too many key variables may make it possible for some adversary to identify individuals within the data set, especially if the data set is combined with auxilliary information (such an action is known as a linkage attack)

So how does one determine how to deal with these problematic key variables? That’s where data anonymization comes in!

Three common techniques in data anonymization are perturbing, generalizing, and suppressing.

Perturbation

When data is perturbed, each observation is slightly modified so that the overall distribution of data is not significantly changed, but the true values of the data itself are unknown. This makes perurbation useful when analysts want to discover overlying trends, without exposing the actual values of information that may be private.

There are two methods of data perturbation.

  1. The first is value perturbation, which is only useful for numeric data. In value perturbation, each element is added with a random value within a user-defined range to distort the original data.

    perturbed value = true value + random noise

    For example, if the true value was 32, and the noise range spanned 1 to 18, then the perturbed value may be anything between 32 and 50.

    Estimations of the true values may be created by subtracting the average random noise value from the perturbed value.

    estimated true value = perturbed value - average random noise

    Continuing the example, if the perturbed value was 47, an analyst attempting to predict the true value would subtract the average random noise of 9 to get 38. 38 is not exactly the true value of 32, but it may or may not be close to it depending on the scale of data.

    If the random noise is small enough compared to the scale of data, the perturbed values may not change significantly, if at all. A larger random noise will make it more difficult to determine the true values.

  2. The second method of perturbation is probability perturbation, in which each element has a user-defined probability of being replaced by another element within the distribution of values.

    For example, suppose an attribute in some data set is color. 50% of elements are blue, 30% are red, and 20% are yellow. Suppose we are perturbing a red element, and we have decided that probability that an element will be perturbed is 25%. This means that there is a 75% chance that the red element will not be perturbed and remain red, and a 25% chance that it will be perturbed and changed to some color from our distribution, maybe even red. So the probability that it will be a color is…

    probability of blue = (perturbation probability)(percentage of blues) = (.25)(.5) = .125

probability of red = (1 - perturbation probability) + (perturbation probability)(percentage of reds) = .75 + (.25)(.3) = .75 + .075 = .825

probability of yellow  = (perturbation probability)(percentage of yellows) = (.25)(.2) = .05

    So if we had a thousand red elements in our data, about 825 of them would remain red, 125 would become blue, and 50 of them would become yellow.

    The higher the perturbation probability, the more likely it is that the perturbed values will be different than the true values, and the harder it will be for analysts to identify the true values in the original data. However, the distribution of values across the data should not change signficantly, regardless of perturbation probability.

Generalization

In generalization, data is not changed but simplified to a broader, more general form. For categorical data, categories are merged together to create broader categories that provide less precise information. For numeric data, numbers are replaced with numeric ranges that represent a possible value within a spectrum of numbers. For example, if a data set contains the values 41, 75, and 99, these values may be replaced with the numeric ranges 41-60, 61-80, and 81-100. Unlike perturbed data, which is by nature incorrect, generalized data does not lose any of its validity. However, because generalized data is always categorical, it may not be desirable when continuous data is preferred. It may also be difficult or impossible to implement in categorical data if only a few categories exist, particularly if the categories are very distinct from each other or if merging the categories would remove any information that they might convey.

Suppression

Finally, when data is suppressed, it is simply removed from the data set. Suppression is typically considered the most extreme out of the three techniques described here, because it grants the greatest anonymity while also removing the greatest amount of information. Both attributes (columns) and observations (rows) may be suppressed.

Typically, attributes are suppressed if they provide too much identifiable information. Identifiers are always suppressed. Key Variables may be suppressed if perturbation or generalization do not sufficiently anonymize the data (i.e., even with perturbation and generalization, it is still possible to identify individuals using some combination of key variables).

Observations may be suppressed if the individual is so unique that it is difficult to anonymize without removing large amounts of information from the data set, or if the information of the individual is particularly sensitive and must be hidden even when anonymized.

Methods

anonymizeR contains several functions to assist in perturbing, generalizing, and suppressing data.

Perturbing Data: perturbation()

For perturbing data, the anonymizeR package contains one method, perturbation(). perturbation() encompasses both the value and probability perturbation methods. perturbation() takes as arguments...
- **x**, a data frame
- **cols**, a vector of column names or indices
- **method**, whether the user would like to use the value perturbation method, the probability perturbation method, or both
- the minimum size of noise (value perturbation). There are two possible arguments for this, only one of which is needed:
    - **minNoise**, the minimum amount of random noise that may be added to any element
    - **minNoiseQuantile**, the minimum amount of random noise that may be added to any element, taken as a quantile of the total range of values for that column rather than as an exact value. For example, if the minNoiseQuantile is .25, and the value at the 25th quantile of the column's distribution is 5, then every perturbed element will be at least 5 greater than it was before perturbation.
- the maximum size of noise (value perturbation). There are two possible arguments for this, only one of which is needed:
    - **maxNoise**, the maximum amount of random noise that may be added to any element
    - **maxNoiseQuantile**, which functions similarly to minNoiseQuantile but for maximum noise rather than minimum
- **probability** the probability of replacing an element (probability perturbation)

Then, for each selected column, the method perturbates the data using the selected technique. If both techniques are selected, then the method uses value perturbation for numeric columns, and probability perturbation for non-numeric columns. 

- For value perturbation, if minNoise and maxNoise are provided, then a random value between minNoise and maxNoise is added to each element. If minNoise or maxNoise is not provided, then the method calculates minNoise or maxNoise using minNoiseQuantile or maxNoiseQuantile, taking that quantile from the data for that column, and substituting that into minNoise or maxNoise.

- For probability perturbation, the method stores the original column. It then randomly decides for each element whether or not to replace it, using the user-provided probability (the default is 50%). If the method decides to replace an element, that element is randomly replaced with some other element from the original column.

Generalizing Data: generalizeNumeric() and generalizeCategorical()

For generalizing data, the anonymizeR package contains a generalizeNumeric() method, which replaces continuous numeric values with categories representing numeric ranges, and a generalizeCategorical() method, which merges categories together. There is also a generalize() method, which is a shortcut for the previous two methods.

generalizeNumeric() takes as arguments...
- **x**, a data frame
- **col**, a column name or index
- How to split the numbers into different numeric ranges. There are two possible arguments for this, only one of which is needed:
    - **splits**, an ordered vector containing the endpoints of the numeric ranges. For example, if splits contains the values 2, 4, and 6, then there would be four numeric ranges: less than 2, between 2 and 4, between 4 and 6, and greater than 6.
    - **numSplits**, the number of splits desired. If this is used instead of splits, the method creates splits such that each numeric range contains an equal number of elements (or as close to equal as possible) by using the quantiles of the original column to determine the splits. For example, if numSplits is 2, and if one third of elements are less than 50, one third of elements are between 50 and 100, and one third of elements are more than 100, then the method creates two splits, 50 and 100.
- **rightClosed**, whether numeric ranges should be closed on the right (Ex: 50 < x <= 100) or not (Ex: 50 <= x < 100). By default FALSE.
    
generalizeNumeric() simply iterates through every element in the column and replaces it with whichever numeric range that element falls in. It then returns the data frame.

For suppressing data, there is a suppress() method, which removes rows and columns from a data frame, and a randomizeCharacters() method, which replaces all the values in a column (including possibly the column name) with random characters - essentially making it impossible to decipher the original values without outright removing the data from the data frame.
